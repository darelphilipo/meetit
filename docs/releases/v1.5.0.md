# v1.5.0 — UI/UX Polish Release

**No API changes. No new endpoints. No new OpenSpec changes.** Pure UI/UX polish that preserves all existing calling conventions, state management, and the 27 OpenSpec items in the roadmap. The app stays as snappy as v1.4.4.

## What changed

### 1. Design system tokens (`public/app.html`)
Added 3 new token families to `:root`:
- **Typography scale** — `--text-xs (11)`, `--text-sm (12)`, `--text-base (14)`, `--text-md (15)`, `--text-lg (17)`, `--text-xl (18)`, `--text-2xl (22)`
- **Motion** — `--t-fast (0.12s)`, `--t-base (0.18s)`, `--t-slow (0.35s)`, `--ease` cubic-bezier
- **Empty state variants** — `.empty-state.compact` for in-overlay empty states (smaller emoji, less padding)

### 2. Hover lift on every interactive element
- `.btn`, `.icon-btn`, `.footer-btn`, `.close-btn`, `#debug-toggle` all get `transform: translate(-1px, -1px)` + slightly larger hard shadow on `:hover`
- Pairs with the existing `:active` translate-down (the "press" effect)
- This is the neo-brutalist "physical" feel

### 3. Keyboard accessibility (`focus-visible`)
- Pink ring (`var(--secondary)`) on every interactive element
- Separate from `:focus` (which fires on click) — only shows on keyboard nav
- Also on `.tab` and on form inputs (yellow primary ring)
- Critical for keyboard / screen-reader users; the app is now Tab-navigable end-to-end

### 4. Reduced-motion support
New `@media (prefers-reduced-motion: reduce)` block:
- Kills all infinite animations (specifically the `.loading-emoji` bounce)
- Shortens all transitions to 0.01ms (no jank, no flicker)
- Respects user OS-level motion preference

### 5. Step dot pulse on multi-step overlays
- New `@keyframes stepPulse` (scale 1 → 1.6 → 1 over 320ms)
- New `pulseDot(id)` helper in `app.ts` triggered by `detailNext/Prev` and `modDetailNext/Prev`
- Visual feedback for "you advanced to step N"
- Centralized in one helper; 6 call sites total

### 6. Fade-once for card shells
- `buildCardShell({ noFade: true })` parameter
- All 6 Prev/Next handlers (home, my-stuff rsvps/events/pitches, mod) pass `noFade: true`
- First render of a card fades in; subsequent navigations are instant
- Fixes the "jittery" feel of tapping Next repeatedly
- Pure additive change: `noFade` defaults to false so first-render behavior is preserved

### 7. Active tab accent
- `.tab.active` now has `box-shadow: inset 0 -4px 0 var(--secondary)` (pink underline)
- Combined with the existing yellow background, gives a clear "you are here" indicator
- No layout shift (inset shadow, not border)

### 8. Empty state consistency
- New `.empty-state.compact` modifier for in-overlay empty states
- The 3 My Stuff empty states (no RSVPs / no pitches / no events) now use the compact class instead of inline `style="font-size:36px"` overrides
- Visually cohesive across the app

### 9. ARIA labels for icon buttons
- All 4 header icon buttons (refresh, create, my-stuff, mod) get `aria-label` + `role="button"` + `tabindex="0"`
- The debug toggle button gets `aria-label="Toggle debug log panel"`
- All 8 static close buttons get `role="button"` + `tabindex="0"` + `aria-label="Close"` via a one-pass `DOMContentLoaded` setup
- Screen readers can now announce every button
