# v1.6.2 — iOS Blank-Webview Fix + CSV Export Safety

Two small hardening fixes that were **prerequisites or had confirmed real-user impact**:
- iOS blank-after-navigate: blocks the bug that surfaces once e13 ships (DM buttons everywhere)
- CSV formula injection: real OWASP-flagged security issue in the attendee export

## What changed

### 1. iOS blank-webview recovery via `visibilitychange`

**Before (≤ v1.6.1):** Calling `navigateTo(url)` (e.g., the share-link button) pushed the user to a native Reddit page. On iOS Safari and the iOS Reddit app, tapping Back often returned to a blank webview — the page chrome was gone, only a white screen remained. The user had to force-quit the Reddit app and reopen the post to recover.

**After (v1.6.2):** A `visibilitychange` listener in `DOMContentLoaded` soft-renders the active surface when the page becomes visible again:

```ts
document.addEventListener("visibilitychange", function () {
  var state = document.visibilityState;
  if (state !== "visible") return;
  if (Date.now() - lastVisAt < 500) return;            // throttle (iOS sometimes double-fires)
  if (visReRenderInProgress) return;                    // idempotency guard
  visReRenderInProgress = true;
  if (mod-screen overlay is active) loadModTab(modTab);
  else if (my-stuff overlay is active) loadMySubmissions();
  else loadHome();
  setTimeout(function () { visReRenderInProgress = false; }, 500);
});
```

**Key design decisions:**
- **Soft-render, not hard re-init.** A full `init()` would wipe open overlays (half-filled forms, open mod dashboard). Soft-render re-fetches data and re-renders the card. Open overlays stay open; the user doesn't lose what they were doing.
- **500ms throttle.** iOS sometimes fires `visibilitychange` twice in quick succession (once on transition, once on focus). The throttle coalesces these.
- **Idempotency guard.** If a re-render is already in flight, the next event is skipped.
- **Per-surface routing.** Routes to the right loader: mod → `loadModTab(modTab)`, my-stuff → `loadMySubmissions()`, otherwise → `loadHome()`. Each loader has its own fetching guard, so re-entry is safe.
- **No CSS changes.** The fix is purely in `app.ts`.

**Why this matters now:** `e13-direct-message-organizer` will add DM buttons for mods and attendees. Every DM button triggers `navigateTo()`. Shipping e13 without this fix would surface the blank-webview bug to every user who ever messages an organizer. Fix it before e13, not after.

### 2. CSV export safety: formula injection + RFC 4180

**Before (≤ v1.6.1):** `onExportAttendees` built CSV with simple string concatenation:

```ts
lines.push(`${a.username},${(a.email || "").replace(/,/g, "")},${(a.phone || "").replace(/,/g, "")}`);
```

Two problems:

1. **CSV formula injection (security).** A username starting with `=`, `+`, `-`, `@`, `\t`, or `\r` is interpreted as a formula when opened in Excel/LibreOffice/Google Sheets. Example: a malicious attendee with username `=cmd|'/c calc'!A1` runs `calc.exe` on the organizer's machine when the CSV is opened. The `.replace(/,/g, "")` strip does nothing to prevent this.
2. **RFC 4180 violation (data corruption).** Any field containing `,` or `"` breaks the CSV. A username like `alice,bob` exports as `alicebob` — silent data loss.

**After (v1.6.2):** A `csvEscape(value)` helper in `src/shared/meetit.ts`:

```ts
export function csvEscape(value: string | null | undefined): string {
  if (value == null) return "";
  let v = String(value);
  if (/^[=+\-@\t\r]/.test(v)) v = "'" + v;  // formula guard
  return `"${v.replace(/"/g, '""')}"`;        // RFC 4180 quote+escape
}
```

`onExportAttendees` now wraps every field in `csvEscape`:

```ts
const header = [csvEscape("Username"), csvEscape("Email"), csvEscape("Phone")].join(",");
const lines = [header];
for (const a of attendees) {
  lines.push([csvEscape(a.username), csvEscape(a.email), csvEscape(a.phone)].join(","));
}
```

**Why the `'` formula guard:** The single quote is the standard "force-text" prefix that tells Excel to treat the cell as a literal string instead of a formula. The `'` is not displayed in the cell; it just suppresses formula evaluation. Quoting alone does NOT prevent formula injection — the formula `=cmd|...` inside `"=cmd|..."` is still a formula. The single-quote guard must come first, then the value is wrapped in double quotes.

**Why include `\t` and `\r` in the guard:** Some Excel versions treat leading tab/CR as formula triggers, especially in combination with other control characters. Including them is the conservative choice.
