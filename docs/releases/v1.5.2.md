# v1.5.2 — Mod Published Card Layout Fix

**No API changes. No new OpenSpec items. No behavior changes.** Fixes the mod dashboard "Published" tab where the Delete Event button was getting pushed outside the card area, and the description body was being wasted.

## The bug

User screenshot showed the mod Published card with:
- 2 stacked rows of action buttons (Details / Attendees on top, Delete Event below)
- The Delete Event button was getting clipped at the bottom of the card
- The body (description box) was effectively empty / unused space

Root cause: `renderModCard` had a 2-row action layout for the published tab, and the body used a paginated description box pattern (from the mod detail overlay) that didn't fit a single-line snippet well.

## The fix

Restructured the Published tab to match the home card pattern (which the user said works well):

### 1. Single row of 3 small action buttons
**Before** (2 rows, ~94px total):
```
[Details →]  [👥 2]    ← row 1
[🗑️ Delete Event]      ← row 2 (clipped at card edge)
```

**After** (1 row, 38px total):
```
[Details →]  [👥 2]  [🗑️]   ← one row, icon button for delete
```

The Delete button uses the new `.btn-icon` class (square 38×38) — context is clear from the card, no label needed. `title="Delete event"` + `aria-label="Delete event"` for accessibility.

### 2. Description snippet body (matches home card)
**Before**: `flex:1` box with `position:relative` containing a `position:absolute` track for pagination (complex; visually empty in screenshot).

**After**: simple scrollable div with `flex:1; min-height:0; overflow-y:auto;` containing the first 120 chars of the description. If empty, shows "No description" in muted italic.

Uses `DESC_PREVIEW_LENGTH` (120) instead of `DESC_SHORT_LENGTH` (100) — same length, more consistent with the home card.

### 3. Removed unused auto-paginate logic
The in-card pagination (28 lines of splitTextToPages / buildModDescPagesHTML / buildModDescNavHTML calls) was removed from `renderModCard`. The state variables `modDescPageIdx` / `modDescTotal` / `modDescFullText` are still updated for consistency with the mod detail overlay (which uses them for the full-page "view mod details" view).

## What was deliberately NOT changed

- The home card (already uses the correct pattern)
- The mod card Pending tab (already has 2 buttons in 1 row: Approve / Decline)
- The mod card Pitches tab (single Dismiss button, already compact)
- The mod event details overlay (full-page view with its own pagination — keep it, it's a different surface)
- API patterns, data shapes, optimistic updates, etc. (per LEARNINGS §40, §41, §42)
