### e13-direct-message-organizer (merged: original e13 + e16)
`sendPrivateMessage({ to: username })` is broken in Devvit Web (LEARNINGS §42). The workaround is `navigateTo("https://www.reddit.com/message/compose?to={username}")` — a public Reddit URL, not an authenticated API call.

Three call sites in one change:
- **Event details (attendee view, step 2)**: "✉️ Message Organizer"
- **Mod pending events card**: "✉️ Message Organizer"
- **Mod pitches card**: "✉️ Message Pitcher"

Empty username guard omits the button (no dead buttons). Strip `u/` prefix, `encodeURIComponent`, log to debug panel with source location.

**Risk documented:** Reddit webview may block `message/compose` URLs as a clickjacking vector. Fallback: `window.open()` in a follow-up if it fails.

### e14-organizer-see-attendees
Currently only mods can see the full attendee list. Organizers submit an event, see "12 RSVPs" as a number, but cannot see **who**.

Pure client-side change: add a "👥 N Attendees" button on each owned published event card in My Stuff. Wire to existing `showModAttendees(id)` (or `/api/export-attendees` directly). Server-side `onExportAttendees` already enforces `isSubmissionOwner` (organizer OR mod), so no server change needed.

Privacy: organizers see Reddit usernames only. No email/phone. Matches the §13 privacy pattern from skills.md.

### e17-event-status-timeline
Replace "⏳ Pending" with relative time + SLA hint:
- `formatRelativeTime(ms)` utility in `meetit.ts` (10 lines, no `date-fns` dependency)
- Buckets: just now / Nm ago / Nh ago / Nd ago / Nw ago / MMM D (absolute date for > 30d)
- Graceful degradation: missing `submittedAt` falls back to `createdAt` or shows "—"

Render in My Stuff cards:
- Pending: "📅 Submitted {label} · Usually reviewed within 48hrs"
- Published: "📈 {n} RSVPs · approved {label}"
- Rejected: "❌ Rejected {label}"

### fix-ios-blank-after-navigate
iOS Safari + iOS Reddit app: webview shows blank page after returning from a `navigateTo` call. Documented in skills.md (Podcast Poster section). The fix: `visibilitychange` listener with 500ms throttle.

- `< 5 min away`: soft render (re-render current tab)
- `> 5 min away`: full re-init (re-fetch home, re-run init)
- In-flight guard prevents double re-init
- Manual test on iOS Safari + iOS Reddit app + Android + desktop

**Why priority 3/5 not 2/5:** the bug currently affects only the share-link button (rare, ~5% of users). It will become more visible once `e13-direct-message-organizer` ships. Fix it before e13, not after.
