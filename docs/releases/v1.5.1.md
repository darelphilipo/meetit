# v1.5.1 — Button Size Standardization

**No API changes. No new OpenSpec items. No behavior changes.** Fixes the "View Details is way bigger than RSVP" visual bug and introduces a section-scoped button size system that makes per-section size management trivial.

## The bug

User screenshot showed that "View Details →" was visibly larger than "🎟️ RSVP" in the home card action row. Root cause:

1. **~40 inline `style="padding:...; font-size:...; margin-top:...; flex:...;"` attributes** had drifted across render functions. Different buttons in the same row used different paddings (8px vs 10px vs 12px) and different font-sizes (11px vs 12px vs 13px vs 14px).
2. **"View Details →" wrapped to 2 lines** at 13px font because the column was too narrow, making the button visibly taller than the shorter "RSVP" button.

## The fix: section-scoped button classes

Every button now uses exactly one of 8 section-scoped classes. Defined once in `app.html` `:root`; no inline style needed.

| Class | Used by | Padding | Font | Min-h |
|---|---|---|---|---|
| `.btn` (default) | Full-width CTAs (Submit Event, RSVP Now) | 10px 16px | 14px | auto |
| **`.btn-action`** | **Action row buttons in card shells (Details, RSVP, Approve, Decline, Update, Leave)** | 8px 6px | 12px | 38px |
| **`.btn-icon`** | **Square icon buttons in action row (Share 📤)** | 8px 10px | 14px | 38px×38px |
| **`.btn-action-full`** | **Full-width single-button rows (Delete Event, Dismiss)** | 10px 8px | 13px | 40px |
| **`.btn-pager`** | **Pagination prev/next (desc/att nav)** | 4px 10px | 11px | 24px |
| **`.btn-compact`** | **Inline buttons in overlay footers (confirm yes/no, RSVP Copy/Done, mod attendees CSV)** | 10px 14px | 13px | auto |
| **`.btn-copy`** | **Inline copy buttons next to links (Google Maps Copy)** | 6px 14px | 12px | auto |
| **`.btn-empty`** | **Buttons inside empty states** | 6px 12px | 12px | 30px |

Adding the right class to the right button in the right section is enough. No more inline style.

## Bug-specific fix: "View Details →" was wrapping

Three changes combine to fix the screenshot:
1. `.btn-action` uses `font-size: 12px` (down from 13px) — more horizontal space for the label
2. `.btn-action` has `white-space: nowrap; overflow: hidden; text-overflow: ellipsis;` — anything that would wrap is now ellipsized (not a wrap)
3. Label shortened from "View Details →" → "Details →" (and from "👁️ View Details →" → "Details →" in mod card; "Update Contact" → "Update" in My Stuff) — all action row labels are now ≤10 chars

The share button got a new `.btn-icon` class so it doesn't expand to fill the row — it's a square 38×38 button next to the flex-1 Details and RSVP buttons.

## What was deliberately NOT changed

- `.btn-sm` and `.btn-inline` classes (left in place; rarely used; not worth removing)
- `style="width:80%;"` on RSVP Now button in details step 4 (centered fullscreen CTA, needs the inline width constraint)
- The 4 outline-style `.tab` headers (separate size system)
- All API call patterns, data shapes, optimistic updates, etc. (per LEARNINGS §0.2, §40, §41, §42)

## Manage sizes per section easily

Per the user's request: "make it such that it's easier to manage the size consistency in each section easily".

The classes are named by **where the button lives** (its section), not by what it does. So if you ever want to:
- Make all card action buttons bigger → edit `.btn-action` in one place
- Make all overlay footer buttons tighter → edit `.btn-compact` in one place
- Make all pagination buttons smaller → edit `.btn-pager` in one place

Compare to ad-hoc utilities like `.btn-pad-sm` (sized-by-property) which would scatter "what size should a mod approve button be?" knowledge across every render function. Section-scoped is one-line changes.
