# v1.5.5 — Horizontal Scrolling for Review Page

**No API changes. No new OpenSpec items. No behavior changes.** Adds horizontal scrolling to the long-text fields on the Submit Event review page (step 5) to match the pattern used elsewhere in the app.

## The fix

The review page (added in v1.5.4) had three fields that wrapped vertically when their content was long:

- **Title** — long event titles wrapped to multiple lines, pushing the meta line down
- **Meta line** — `📅 DATE at TIME · 📍 LOCATION · 🏷️ CATEGORY` could be very long, wrapped to multiple lines
- **Description** — used `word-break: break-word; white-space: pre-wrap;` which broke long URLs or copy-pasted text at any character, making them hard to read

## Changes (1 file: `public/app.html`)

### Title
```css
/* Before: text wraps to multiple lines, takes vertical space */
#event-review-title-preview { font-size: 17px; ... }

/* After: single line, horizontal scroll if too long */
#event-review-title-preview {
  white-space: nowrap;
  overflow-x: auto;
  overflow-y: hidden;
  -webkit-overflow-scrolling: touch;
  padding-bottom: 2px;  /* room for the scrollbar */
}
```

### Meta line
Same pattern as title — `white-space: nowrap; overflow-x: auto; overflow-y: hidden; -webkit-overflow-scrolling: touch; padding-bottom: 2px;`

### Description
```css
/* Before: breaks long unbroken strings at any character */
#event-review-desc-preview {
  word-break: break-word;
  white-space: pre-wrap;
  overflow-y: auto;
  max-height: 200px;
}

/* After: preserves long unbroken strings, scrolls in BOTH directions */
#event-review-desc-preview {
  white-space: pre;            /* preserves whitespace + long strings */
  overflow: auto;              /* scrolls vertically AND horizontally */
  min-width: 0;                /* allow the box to shrink (key for overflow-x to work in flex) */
  max-width: 100%;             /* constrain to parent */
  max-height: 200px;
  -webkit-overflow-scrolling: touch;
}
```

The `min-width: 0` trick is the standard fix for horizontal scrolling in a flex child — without it, the box would refuse to scroll horizontally because flex items default to `min-width: auto` and would rather overflow than scroll.

## What was deliberately NOT changed

- API patterns, data shapes, validation rules (per LEARNINGS §40, §41, §42)
- The horizontal pagination pattern used by mod details / mod attendees (LEARNINGS §37) — that uses prev/next buttons; the review page has no next page, so per-field horizontal scroll is more appropriate here
- The 4 other overlays (my-stuff, details, mod, mod-event-details)
- `submitEvent()` function (unchanged)
