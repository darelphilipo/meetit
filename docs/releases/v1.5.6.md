# v1.5.6 — Review Page: Description Box Uses Available Space

**No API changes. No new OpenSpec items. No behavior changes.** Fixes the description box on the Submit Event review page (step 5) that was rendering as a tiny 1-line strip despite the v1.5.5 horizontal-scroll changes.

## The bug (user screenshot)

After v1.5.5 added horizontal scrolling to title, meta, and description on the review page, the description box still showed only ~1 line of text and the rest of the overlay body was empty yellow space:

- Title showed as a single line (works correctly, but no way to know it scrolls horizontally)
- Meta line was clipped at the visible edge ("good nigh")
- Description box was only 30px tall, showing one line, no obvious way to scroll

Root cause: the description box had `max-height: 200px; overflow: auto` but with `white-space: pre`, the text stayed on one line and the box showed only what fit horizontally. The vertical scrollbar that would normally appear never did, because there was only one visual line.

## The fix (1 file: `public/app.html`)

Restructured step 5 to be a **full-height flex container** with the description box **filling the remaining vertical space** between the meta line and the helper text.

### Step 5 outer (now flex column)
```css
.overlay-body#event-step-5 {
  display: flex;
  flex-direction: column;  /* fills available height */
}
```

### Inner review box (fills available space)
```css
{
  display: flex;
  flex-direction: column;
  flex: 1;          /* fills remaining vertical space */
  min-height: 0;    /* allow shrinking below content size */
}
```

### Title
```css
/* Before */
white-space: nowrap; overflow-x: auto;  /* forced single line, hidden horizontal scroll */

/* After */
overflow-wrap: break-word;  /* wraps on word boundaries, fully visible */
```

### Meta line
```css
/* Before */
white-space: nowrap; overflow-x: auto;  /* same issue */

/* After */
overflow-wrap: break-word;  /* wraps on word boundaries */
```

### Description
```css
/* Before */
max-height: 200px;
overflow: auto;
white-space: pre;            /* no wrap, hidden content */

/* After */
flex: 1;                     /* fills remaining space */
min-height: 0;
overflow: auto;              /* vertical scroll IS discoverable */
white-space: pre-wrap;       /* wraps on spaces + newlines */
word-break: break-word;      /* breaks long unbroken strings (URLs) */
```

The `flex: 1; min-height: 0;` on the description is the key — it makes the box grow to fill the space between the meta line and the helper text. With `min-height: 0` (the standard flex shrinking trick), the box can shrink below its content height, so the `overflow: auto` vertical scroll works naturally.

## What this looks like

- **Title**: wraps to multiple lines if too long (visible in full)
- **Meta**: wraps to multiple lines if too long (visible in full)
- **Description**: uses all available vertical space (~300-400px on a phone); vertical scrollbar appears naturally on iOS Safari when content overflows; long unbroken strings (URLs) break and don't force horizontal scroll
- **Helper text**: stays pinned at the bottom (`flex-shrink: 0`)
