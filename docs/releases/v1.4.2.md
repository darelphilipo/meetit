# v1.4.2 — Project Hygiene: docs/ Folder Structure

**No code changes, no behavior changes.** Pure project organization.

## What changed

Repo root was cluttered with 3 archived .md files + 2 release notes. They're now organized in a `docs/` tree.

### Before
```
.
├── AUDIT.md
├── BUG_REGISTRY.md
├── ENHANCEMENTS.md
├── LEARNINGS.md
├── README.md
├── TEST_CASES.md
├── .opencode/
│   └── release-notes-v1.4.0.md
│   └── release-notes-v1.4.1.md
└── ...
```

### After
```
.
├── LEARNINGS.md        (100KB, platform knowledge base — keep at root)
├── README.md
├── TEST_CASES.md
├── docs/
│   ├── README.md       (new — index of all documentation)
│   ├── archive/        (historical, do not edit)
│   │   ├── AUDIT.md
│   │   ├── BUG_REGISTRY.md
│   │   └── ENHANCEMENTS.md
│   └── releases/
│       ├── v1.4.0.md
│       └── v1.4.1.md
└── .opencode/
    ├── opencode.json
    └── commands/  skills/   (no release notes anymore)
```

## Why

- Root is now just essentials: `LEARNINGS.md`, `README.md`, `TEST_CASES.md`, `LICENSE`, `devvit.json`, `package.json`, `tsconfig.json`, config files.
- `docs/README.md` is the single entry point to all documentation.
- Historical files (archive/) and release notes (releases/) are clearly separated from active content.
- Future archived material goes into `docs/archive/` following the same pattern.
