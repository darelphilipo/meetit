### fix-prefill-loading-stuck
Two loading flags can get stuck `true` if async work throws:
- `prefillLoading` in `prefillOrganizer` (`app.ts:1766`) — no try/finally; flag stays true on fetch error
- `myStuffLoading` in `loadMySubmissions` (`app.ts:468`) — reset is after try/catch; if catch block throws, flag stays stuck

**Fix:** wrap both in try/finally.

### fix-approve-lock-cleanup
`onApproveEvent` (`server.ts:444-449`) acquires a distributed lock with 10s TTL but never explicitly releases it. If `hSet` or `hDel` throws, the lock waits 10s to expire.

**Fix:** wrap hSet+hDel in try/finally with `redis.del(lockKey)`. 10s TTL remains as safety net.

### fix-desc-pagination-shared-state
`descFullText`, `descPageIdx`, `descPageTotal` are keyed by raw event ID. Both user detail and mod detail overlays use the same keys. If a mod opens mod detail for an event the user is viewing, pagination state corrupts.

**Fix:** introduce `modDesc*` namespaced records; mod handlers use them, user handlers unchanged.

### fix-desc-pagination-font-size
`splitTextToPages` (`app.ts:691-710`) hardcodes `font-size:15px` in the measurement div. But mod cards render at 14px and My Stuff at 13px. Page splits are slightly wrong (slightly more text per page than fits).

**Fix:** add `fontSize` parameter to `splitTextToPages`; update 5 call sites with the correct size (15/14/13).

### fix-bug2-cron-reminder-retry (regression note added)
Current code (`server.ts:777-778`) sets `remindedKey` BEFORE the reminder post, so a failed post leaves the flag set and the reminder will never retry. This is the exact failure mode the change is supposed to fix. **Implementation must move lines 777-778 INSIDE the try block, after the successful `submitCustomPost` call.**
