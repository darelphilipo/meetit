# calendar-export Specification

## Purpose
Allows users to add a Meetit event to their Google Calendar via a link embedded in the automated reminder and share posts. The link opens Google Calendar's "Add Event" page with the event's title, date, time, location, description, and Google Maps URL prefilled.

## Background: why links in posts, not buttons in the app
The original plan was to add a "📅 Add to Calendar" button inside the Devvit webview (event details + RSVP success card). Manual testing on r/meetup_hub2_dev revealed that opening external URLs from a Devvit webview crashes the post iframe with the error "this post failed to load because of an unknown error." Root cause: the Devvit webview sandbox lacks the `allow-popups` and `allow-top-navigation` flags needed for `window.open()` or `window.location.href`. `navigateTo` (the Devvit Web global) only works for Reddit-internal URLs.

The fix is to put the calendar link in the post body instead of a button. Reminder and share posts are plain text posts (not custom app posts) that render in the native Reddit client, where external links work fine and show Reddit's native external-link confirmation dialog before opening Google Calendar. This also reaches more users — anyone reading the post, not just app users.

## Requirements

### Requirement: Reminder posts include an "Add to Google Calendar" link
The reminder post body SHALL include a `## 📅 [Add to Google Calendar](...)` link section. The link SHALL be placed after the existing `## 🗺️ [Open in Google Maps]` section (when present) and before the description and attendee sections. The link is omitted entirely when the event has no `date` or the date string is malformed.

#### Scenario: Reminder post for event with date and time
- **WHEN** the CRON builds a reminder body for an event with date "2026-06-25" and time "22:50" in timezone "+05:30"
- **THEN** the body contains `## 📅 [Add to Google Calendar](https://calendar.google.com/calendar/render?action=TEMPLATE&text=...&dates=20260625T172000Z%2F20260625T182000Z&location=...&details=...)`
- **AND** the dates parameter is in UTC (local time converted using the configured timezone)
- **AND** the end time is start + 1 hour

#### Scenario: Reminder post for event without a date
- **WHEN** the reminder body is built for an event with an empty `date` field
- **THEN** the body does NOT contain any "Add to Google Calendar" link
- **AND** no `## 📅 [Add to Google Calendar]` section is rendered

### Requirement: RSVP share posts include an "Add to Google Calendar" link
The RSVP share post body SHALL include a `## 📅 [Add to Google Calendar](...)` link section. The link SHALL be placed after the existing `## 🗺️ [Open in Google Maps]` section (when present) and before the description section. The link is omitted entirely when the event has no `date`.

#### Scenario: Share post for event with full data
- **WHEN** the user shares an event with date, time, location, and mapUrl
- **THEN** the share post body contains the `## 📅 [Add to Google Calendar](...)` link after the Maps link
- **AND** the link's `details` parameter includes the mapUrl (separated by a newline + 🗺️ emoji)

#### Scenario: Share post for event without a date
- **WHEN** the share post is built for an event with an empty `date` field
- **THEN** the body does NOT contain any "Add to Google Calendar" link

### Requirement: Calendar URL is generated by a shared pure function
The system SHALL use a shared `buildGoogleCalendarUrl(event, timezone)` pure function in `src/shared/meetit.ts` to generate the calendar URL. The function SHALL be used by both `buildReminderBody` and `buildRsvpShareBody`. It SHALL accept a `MeetitEvent` and a timezone offset string (e.g., `"+05:30"`), convert the local date+time to UTC, format the `dates` parameter as `YYYYMMDDTHHMMSSZ/YYYYMMDDTHHMMSSZ` (start + 1 hour), and return an empty string when the date is missing or malformed.

#### Scenario: Function returns URL with all event fields
- **WHEN** `buildGoogleCalendarUrl` is called with a complete event
- **THEN** it returns a URL with `action=TEMPLATE`, `text`, `dates`, `location`, and `details` parameters
- **AND** the `details` parameter includes the mapUrl appended on a new line when present

#### Scenario: Function returns empty string for missing date
- **WHEN** `buildGoogleCalendarUrl` is called with an empty or malformed date
- **THEN** it returns `""` rather than a bogus URL

#### Scenario: Function defaults timezone to +05:30 when not provided
- **WHEN** `buildGoogleCalendarUrl` is called with an empty timezone string
- **THEN** it uses "+05:30" as the timezone

## Out of scope (explicitly deferred)

- **In-app "Add to Calendar" button** — original e31 plan, REMOVED because external navigation from Devvit webview crashes the post iframe
- **`.ics` file clipboard copy** — same crash reason; would require external download
- **Apple Calendar (`webcal://`) support** — not relevant since calendar is now in posts (Reddit links work natively)
- **Outlook / Yahoo direct links** — can be added later as additional sections in the post body
- **Calendar export from My Stuff → RSVPs card** — events don't have start times in My Stuff; the post-body approach reaches users wherever they encounter the event
